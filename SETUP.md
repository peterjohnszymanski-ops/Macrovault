# MacroVault — platform setup notes

This repo ships `lib/`, `pubspec.yaml`, tests, and docs. The native `ios/` and
`android/` folders are generated by `flutter create` (see README). After that
one-time generation, apply the permission keys below — the app uses the camera
(barcode scan + photos), the photo library, and biometrics (Vault lock), and
iOS/Android will hard-crash on first use if the usage strings are missing.

## iOS — `ios/Runner/Info.plist`

Add these keys inside the top-level `<dict>`:

```xml
<key>NSCameraUsageDescription</key>
<string>MacroVault uses the camera to scan barcodes and take progress photos.</string>

<key>NSPhotoLibraryUsageDescription</key>
<string>MacroVault can import progress photos from your library. Imported photos are copied privately and stripped of location data.</string>

<key>NSFaceIDUsageDescription</key>
<string>MacroVault uses Face ID to lock your private Progress Vault.</string>
```

Minimum deployment target: iOS 13.0 (SQLCipher + mobile_scanner). In
`ios/Podfile` set: `platform :ios, '13.0'`.

## Android — `android/app/src/main/AndroidManifest.xml`

Add above the `<application>` tag:

```xml
<uses-permission android:name="android.permission.CAMERA" />
<uses-feature android:name="android.hardware.camera" android:required="false" />
```

`minSdkVersion` should be 21+ (set in `android/app/build.gradle`). For
`local_auth`, the host `MainActivity` must extend `FlutterFragmentActivity`
instead of `FlutterActivity`.

## Verify

```bash
flutter pub get
flutter analyze
flutter test          # runs the pure-Dart domain tests (trend, targets, weekly metrics)
flutter run
```
